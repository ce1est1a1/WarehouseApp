# Тестовые сценарии — Складская система

Документ описывает смоук-проверки и ручные тестовые сценарии для приложения.
Автоматизированные смоук-тесты и юнит-тесты находятся в проекте `WarehouseApp.Tests`
(`Smoke/SmokeTests.cs`, `Unit/*`). Запуск: `dotnet test`.

Учётные данные по умолчанию: логин `admin`, пароль `admin` (роль «Администратор»).

---

## 1. Смоук-проверки (быстрый прогон перед демонстрацией)

| № | Проверка | Ожидаемый результат |
|---|----------|---------------------|
| S1 | Запуск приложения | Открывается окно входа, без ошибок |
| S2 | Вход admin/admin | Открывается главное окно, видны вкладки: Каталог, Поставки, История отгрузок, Отчётность, Тепловая карта |
| S3 | Открыть «Тепловая карта» | Рисуется сетка ячеек A‑01…F‑08, легенда цветов читается, цвета совпадают с подписями |
| S4 | Открыть «Поставки» → создать поставку | Форма открывается, поля и кнопки на месте, дата отображается корректно |
| S5 | Открыть «Проверка и логистика» при оформлении отгрузки | Окно листается до конца, виден блок погоды и рекомендация по страховке |
| S6 | Открыть «Отчётность» | Таблицы отгрузок/списаний строятся, кнопка «Экспорт PDF» доступна |

Автоматический аналог S1–S6 — `SmokeTests.Smoke_AppServices_BootsIocContainerAndResolvesAllServices`
и `SmokeTests.Smoke_SupplyHeatmapShipmentReport_HappyPath`.

## 2. Авторизация и роли

**TC‑AUTH‑1. Успешный вход администратора**
- Предусловия: приложение запущено.
- Шаги: ввести `admin` / `admin`, нажать «Войти».
- Ожидаемо: вход выполнен, доступны все вкладки.

**TC‑AUTH‑2. Неверный пароль**
- Шаги: ввести `admin` / `wrong`.
- Ожидаемо: вход отклонён, показано понятное сообщение, данные формы не теряются.

---

## 3. Каталог и поставки

**TC‑SUP‑1. Регистрация поставки**
- Предусловия: в каталоге есть хотя бы один товар.
- Шаги: «Поставки» → создать поставку → выбрать товар, кол‑во 10, цену, срок реализации → «Сохранить».
- Ожидаемо: поставка создана; остаток товара вырос на 10; создана партия; товар появился на тепловой карте.
- Автотест: `SupplyServiceTests.CreateSupply_ValidSupply_*`, `*_PlacesProductInFreeWarehouseCell`.

**TC‑SUP‑2. Импорт позиций из CSV**
- Шаги: в форме поставки нажать «Импорт из файла», выбрать `.csv` (строки: артикул/название; кол‑во; цена; срок).
- Ожидаемо: позиции добавляются в таблицу; некорректные строки пропускаются; показано число импортированных.

**TC‑SUP‑3. Срок реализации не позже даты поставки**
- Шаги: указать срок реализации = дате поставки.
- Ожидаемо: сохранение отклонено с сообщением «срок реализации должен быть позже даты поставки».
- Автотест: `SupplyServiceTests.CreateSupply_SaleDeadlineNotAfterSupplyDate_*`.

**TC‑SUP‑4. Проверка поставщика по ИНН**
- Шаги: ввести ИНН поставщика, нажать «Проверить».
- Ожидаемо: чистый ИНН → «рисков не найдено, можно работать»; рисковый (`7700000001/2/3`) → «найден риск», при сохранении запрос подтверждения.

## 4. Отгрузка, проверка получателя, логистика

**TC‑SHIP‑1. Оформление отгрузки**
- Шаги: создать отгрузку, добавить товар, открыть «Проверка и логистика», указать получателя и город доставки, «Сохранить» → «отгрузить».
- Ожидаемо: отгрузка создана; остаток товара уменьшился; запись появилась в «История отгрузок» и «Отчётность».

**TC‑SHIP‑2. Недостаточный остаток**
- Шаги: запросить к отгрузке больше единиц, чем на складе.
- Ожидаемо: отгрузка отклонена с сообщением о нехватке товара.

**TC‑CHK‑1. Проверка получателя по ИНН**
- Шаги: ввести ИНН получателя, «Проверить».
- Ожидаемо: три блока (налоги/банкротство/директор) со статусами; итоговый статус «можно продолжить сделку» либо «требуется решение администратора».
- Автотест: `CounterpartyServiceTests.*`.

**TC‑INN‑1. Валидация ИНН**
- Шаги: ввести `abc`, `12345`, пусто.
- Ожидаемо: проверка не выполняется, показано пояснение (только цифры / 10 или 12 цифр / введите ИНН).

**TC‑GEO‑1. Подсказки города и маршрут**
- Шаги: в поле «Город доставки» начать вводить название города.
- Ожидаемо: выпадает список городов; после выбора рассчитывается реальное расстояние от склада (Москва) до города.

**TC‑GEO‑2. Город не распознан**
- Шаги: ввести бессмысленный текст.
- Ожидаемо: блок маршрута/погоды показывает понятное предупреждение «уточните адрес», приложение не падает.
- Автотест: `WeatherServiceTests.GetForecast_WhenCoordinatesAndCityCannotBeResolved_*`.

**TC‑WTHR‑1. Прогноз погоды через 2 дня**
- Шаги: выбрать город, дождаться загрузки прогноза.
- Ожидаемо: показаны температура, ощущается как, влажность, ветер, состояние; строка прогноза видна целиком (не обрезается).

**TC‑REC‑1. Рекомендация по страховке (только жара/холод)**
- Шаги: выбрать город с высоким погодным риском, обновить прогноз.
- Ожидаемо: «Страховка груза» = «Рекомендуется» при высоком риске, иначе «Не требуется». Название товара и характеристики не влияют.
- Автотест: `LogisticsServiceTests.*`, `SmokeTests.Smoke_CounterpartyCheckAndInsuranceRecommendation`.

## 5. Тепловая карта склада

**TC‑HM‑1. Цвет по сроку хранения**
- Шаги: открыть карту в режиме «По сроку хранения».
- Ожидаемо: цвет ячейки соответствует легенде: >90 — зелёный, 31–90 — жёлтый, 8–30 — оранжевый, <8 — красный, пусто — серый. Цвета совпадают с подписями легенды.
- Автотест: `HeatmapServiceTests.GetCells_*`.

**TC‑HM‑2. Появление нового товара на карте**
- Шаги: оформить поставку нового товара, открыть/обновить карту.
- Ожидаемо: товар занимает свободную ячейку и отображается (а не пропадает).
- Автотест: `SupplyServiceTests.CreateSupply_ForProductWithoutCell_PlacesProductInFreeWarehouseCell`.

**TC‑HM‑3. Переключение режима «По скорости движения»**
- Шаги: переключить режим.
- Ожидаемо: легенда меняется на низкая/средняя/высокая; цвета пересчитываются по отгрузкам за период.
- Автотест: `HeatmapServiceTests.GetCells_MovementUnitsWithinPeriod_*`.

**TC‑HM‑4. Данные выбранной ячейки**
- Шаги: кликнуть по ячейке.
- Ожидаемо: в боковой панели — код, товар, остаток, срок хранения, скорость движения; для пустой — «Ячейка свободна».

---

## 6. Отчётность

**TC‑RPT‑1. Отчёт за период**
- Шаги: «Отчётность» → выбрать период → «Показать».
- Ожидаемо: таблицы «Отгрузки» и «Списания» с итогами.

**TC‑RPT‑2. Экспорт в PDF**
- Шаги: «Экспорт PDF», выбрать путь.
- Ожидаемо: создаётся PDF с кириллицей; пустые разделы не выводятся; при отсутствии данных — сообщение «Нет данных для экспорта».

---

## 7. Надёжность

**TC‑NET‑1. Недоступность внешних сервисов**
- Шаги: отключить интернет, открыть логистику/погоду.
- Ожидаемо: приложение продолжает работать, показывает приблизительные данные/предупреждение, не падает.
